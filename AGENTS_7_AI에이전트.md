# 민주AI 7개 AI 에이전트 상세 설명

> 7개 AI 에이전트가 24시간 365일 자동으로 정당을 운영합니다.
> 각 에이전트는 전문 역할을 맡아 독립적으로 작동하면서, 오케스트레이터의 조율 아래 협업합니다.

---

```
                    ┌──────────────────┐
                    │  오케스트레이터    │
                    │  (총사령관/AI대표) │
                    └────────┬─────────┘
                             │ 조율·지시
          ┌──────────┬───────┼───────┬──────────┬──────────┐
          ▼          ▼       ▼       ▼          ▼          ▼
     ┌────────┐ ┌────────┐ ┌────┐ ┌──────┐ ┌────────┐ ┌────────┐
     │정책연구│ │정책분석│ │상담│ │마케팅│ │데이터  │ │의정감시│
     │Research│ │Policy │ │Bot │ │SNS   │ │분석    │ │Monitor │
     └────────┘ └────────┘ └────┘ └──────┘ └────────┘ └────────┘
```

## 1. 🎯 오케스트레이터 (Orchestrator Agent)

### 역할: 총사령관 / AI 당대표

전체 6개 에이전트를 통합 관리하는 "AI 당대표"입니다.
회사의 CEO처럼 각 부서(에이전트)가 잘 돌아가도록 지휘합니다.

### 핵심 기능
| 기능 | 설명 |
|------|------|
| 스케줄 관리 | 주간 연구 일정, SNS 게시 타이밍, 데이터 수집 주기 자동 조율 |
| 시스템 조율 | 정책연구 결과 → 마케팅팀 전달 → SNS 게시 같은 에이전트 간 협업 |
| 대표 브리핑 | 당 전체 활동 현황을 종합 리포트로 생성 |
| 우선순위 결정 | 긴급 이슈 발생 시 어떤 에이전트가 먼저 대응할지 결정 |
| 상태 모니터링 | 각 에이전트의 작동 상태(LIVE/ERROR) 실시간 확인 |

### 기술 구현
- **파일**: `agents/orchestrator.py`
- **AI 모델**: GPT-4o mini
- **API**: `/api/dashboard` (전체 현황), `/api/agents/activities` (활동 로그)

### 작동 예시
```
[09:00] 오케스트레이터: 주간 정책 연구 시작 지시 → 정책연구팀
[09:30] 오케스트레이터: 연구 결과 수신 → 마케팅팀에 전달
[10:00] 오케스트레이터: SNS 게시물 생성 확인 → 승인
[12:00] 오케스트레이터: 시민 상담 23건 처리 확인
[18:00] 오케스트레이터: 일간 브리핑 생성 → 대시보드 업데이트
```

## 2. 🔬 정책 연구 에이전트 (Policy Research Agent)

### 역할: 싱크탱크 / 주간 연구원

매주 1개 주제를 자동 선정해서 깊이 있는 정책 연구를 수행합니다.
1년 52주 = 52개 주제를 체계적으로 다루는 **정책 온톨로지** 시스템의 핵심입니다.

### 핵심 기능
| 기능 | 설명 |
|------|------|
| 주제 자동 선정 | 52주 로드맵에 따라 매주 주제 자동 결정 |
| 글로벌 비교 | 한국 vs 미국 vs 일본 vs EU 등 국제 비교 분석 |
| 정책안 생성 | 연구 결과 기반 실현 가능한 정책안 자동 작성 |
| 온톨로지 저장 | 연구 결과를 지식 그래프에 축적 |
| 아카이브 관리 | 과거 연구 결과 열람 및 검색 |

### 기술 구현
- **파일**: `agents/policy_research_agent.py`
- **AI 모델**: GPT-4o mini
- **API**: `/api/research/run` (연구 실행), `/api/research/list` (목록), `/api/research/{id}` (상세)
- **데이터**: Supabase `policy_research` 테이블

### 52주 연구 주제 순환
```
Q1 (1~13주): 경제·고용·산업
  1주: 최저임금    2주: 청년고용   3주: 스타트업 생태계
  4주: 부동산      5주: 가계부채   6주: 디지털경제 ...

Q2 (14~26주): 교육·복지·보건
  14주: 공교육     15주: 대학교육  16주: 평생교육
  17주: 기초생활   18주: 노인복지  19주: 장애인 ...

Q3 (27~39주): 환경·에너지·기술
  27주: 탄소중립   28주: 재생에너지 29주: AI 규제
  30주: 미세먼지   31주: 물관리    32주: 우주산업 ...

Q4 (40~52주): 외교·안보·거버넌스
  40주: 한미동맹   41주: 남북관계  42주: 통일정책
  43주: 국방개혁   44주: 사이버안보 45주: 선거제도 ...
```

## 3. 📋 정책 분석 에이전트 (Policy Agent)

### 역할: 정책 평가관 / 시민 제안 분석가

시민들이 제안한 정책을 AI가 객관적으로 분석·평가합니다.
감정이나 이념이 아닌, 데이터와 논리 기반의 공정한 분석을 수행합니다.

### 핵심 기능
| 기능 | 설명 |
|------|------|
| 제안 접수 | 시민 정책 제안을 자동으로 접수·분류 |
| 실현가능성 분석 | 예산, 법률, 행정 측면에서 실현 가능성 AI 판단 |
| AI 평가 | 찬성/반대/보류 비율로 객관적 평가 |
| 통계 집계 | 분야별 제안 수, 시민 관심사 트렌드 분석 |
| 유사 정책 연결 | 과거 연구 결과와 시민 제안을 자동 연결 |

### 기술 구현
- **파일**: `agents/policy_agent.py`
- **AI 모델**: GPT-4o mini
- **API**: `/api/policies/analyze` (분석), `/api/policies/list` (목록)

### 분석 프로세스
```
시민 제안 입력
    ↓
[1단계] 분류: 어떤 정책 분야인지 자동 분류
    ↓
[2단계] 유사도 검색: 기존 연구/정책과 유사한 것이 있는지 확인
    ↓
[3단계] AI 분석: 실현가능성, 예산, 법적 타당성 분석
    ↓
[4단계] 평가 리포트: 종합 평가 결과 시민에게 반환
    ↓
[5단계] 투표 진행: 다른 시민들의 찬반 투표
```

## 4. 💬 시민 상담 에이전트 (Support Agent)

### 역할: 24시간 AI 챗봇 / 민원 창구

시민들의 모든 질문에 24시간 쉬지 않고 답변합니다.
단순 FAQ부터 복잡한 정책 질문까지 처리합니다.

### 핵심 기능
| 기능 | 설명 |
|------|------|
| 24/7 챗봇 | 새벽 3시에 질문해도 즉시 답변 |
| FAQ 자동 응답 | "AI 정당이 뭐예요?" 같은 자주 묻는 질문 즉답 |
| 문의 자동 분류 | 정책 질문 → 정책팀, 기술 문의 → 관리자 자동 라우팅 |
| 솔루션 엔진 연동 | 정책 질문 자동 감지 → 근거 기반 솔루션 제공 |
| 대화 컨텍스트 유지 | 이전 대화를 기억하고 맥락에 맞는 답변 |

### 기술 구현
- **파일**: `agents/support_agent.py`
- **AI 모델**: GPT-4o mini
- **API**: `/api/chat` (대화), `/api/solution` (솔루션 엔진)
- **캐싱**: 24시간 TTL로 동일 질문 캐시 (비용 50% 절감)

### 질문 유형별 처리
```
일반 질문: "AI 정당이 뭐예요?"
  → FAQ 답변 즉시 제공

정책 질문: "청년 주거 문제 어떻게 해결하나요?"
  → 솔루션 엔진 모드 자동 전환
  → 근거 기반 해결책 + 데이터 + 해외 사례 제시

기술 질문: "앱이 안 돼요"
  → 기술 지원 안내 + 관리자 알림

의견 제출: "이런 정책이 좋을 것 같아요"
  → 정책 분석 에이전트로 자동 전달
```

### 기존 정당과의 차이
```
기존 정당: 전화 → 대기 → "점심시간입니다" → 다시 전화 → 담당자 부재
민주AI:    앱 열기 → 질문 입력 → 3초 내 AI 답변 → 24시간 가능
```

## 5. 📱 SNS 마케팅 에이전트 (Marketing Agent)

### 역할: 홍보팀장 / 콘텐츠 크리에이터

정책 연구 결과를 시민에게 쉽고 매력적으로 전달하는 콘텐츠를 자동 생성합니다.

### 핵심 기능
| 기능 | 설명 |
|------|------|
| 콘텐츠 자동 생성 | 정책 연구 결과 → SNS용 게시물 자동 생성 |
| 해시태그 최적화 | 트렌드 분석으로 최적의 해시태그 추천 |
| 성과 분석 | 어떤 콘텐츠가 반응이 좋은지 AI 분석 |
| 톤&매너 조절 | 트위터는 짧게, 블로그는 길게 자동 최적화 |
| 캠페인 관리 | 주간/월간 홍보 캠페인 자동 기획 |

### 기술 구현
- **파일**: `agents/marketing_agent.py`
- **AI 모델**: GPT-4o mini
- **API**: `/api/marketing/stats` (성과), `/api/marketing/hashtags` (해시태그)
- **캐싱**: 12시간 TTL로 해시태그 캐시 (비용 절감)

### 콘텐츠 생성 예시
```
[입력] 주간 연구: "청년 주거 정책 - 공공임대 30만호 확대 제안"

[출력] SNS 게시물 5종:

1. 트위터용 (280자):
   "청년 주거비 부담률 42%, OECD 평균보다 14%p 높습니다.
   민주AI 솔루션: 공공임대 30만호 + 주거수당 월 20만원
   #청년주거 #민주AI #AI정당 #주거정책"

2. 인스타그램용 (카드뉴스 텍스트):
   [슬라이드1] 청년 10명 중 4명이 월급의 40% 이상을 집세로...
   [슬라이드2] 해외는 어떻게? 싱가포르 HDB, 오스트리아 빈...
   [슬라이드3] 민주AI 제안: 3단계 청년 주거 안정 플랜...

3. 블로그용 (장문):
   [2,000자 심층 분석 글]

4. 유튜브 숏츠 대본:
   [60초 스크립트]

5. 카카오톡 메시지:
   [간결한 요약 + 링크]
```

## 6. 📊 데이터 분석 에이전트 (Analytics Agent)

### 역할: 선거 전략가 / 데이터 사이언티스트

여론, 트렌드, 선거 데이터를 AI가 분석합니다.
감이 아닌 데이터로 전략을 수립합니다.

### 핵심 기능
| 기능 | 설명 |
|------|------|
| 지지율 예측 | 현재 활동 기반 예상 지지율 시뮬레이션 |
| 트렌드 분석 | 시민 관심사 실시간 파악 |
| 선거 시뮬레이션 | 지역구별 출마 시 예상 득표율 예측 |
| 경쟁 분석 | 다른 정당의 정책 비교 분석 |
| 대시보드 | 주요 지표를 한눈에 보는 시각화 |

### 기술 구현
- **파일**: `agents/analytics_agent.py`
- **AI 모델**: GPT-4o mini
- **API**: `/api/analytics/stats` (통계), `/api/analytics/trends` (트렌드)

### 분석 예시
```
선거 시뮬레이션 결과:

서울 노원구갑:
├── 현역: 우원식 (무소속, 5선) - 예상 38%
├── AI정당 후보: 예상 22% (성장 가능)
├── 기타: 예상 40%
├── 승리 조건: 현역 약점(법안 0건) 집중 부각
└── 판정: 도전적이나 가능성 있음

주간 트렌드:
├── 1위: 부동산 정책 (관심도 ↑23%)
├── 2위: AI 규제 (관심도 ↑18%)
├── 3위: 교육 개혁 (관심도 ↑12%)
└── 시사점: 다음 주 연구 주제를 부동산으로 조정 권고
```

### 기존 정당과의 차이
```
기존 정당: 여론조사 업체에 수천만원 의뢰 → 2주 후 결과
민주AI:    AI가 실시간 데이터 분석 → 즉시 인사이트 제공 → 비용 0원
```

## 7. 🔍 의정 감시 에이전트 (Monitoring Agent) — v2.0

### 역할: 국회 감시관 / 의원 성적표

**국회 열린데이터 API**를 연동하여 현역 국회의원의 의정활동을 실시간 추적합니다.
가상 데이터가 아닌 **실제 국회 공식 데이터**를 사용합니다.

### 핵심 기능
| 기능 | 설명 |
|------|------|
| 실시간 데이터 수집 | 국회 열린데이터 API에서 의원 정보 + 법안 데이터 수집 |
| 의원 검색 | 이름, 지역구, 정당으로 의원 검색 |
| 법안 발의 추적 | 의원별 발의/공동발의 법안 수 집계 |
| 취약 지역구 발굴 | 활동 저조 의원의 지역구를 자동 발굴 |
| AI 의원 분석 | GPT-4o mini가 의원 의정활동 종합 평가 |
| AI 공략 전략 | 지역구별 출마 전략 자동 생성 |
| 종합 리포트 | 전체 국회 의정활동 분석 리포트 |

### 기술 구현
- **파일**: `agents/monitoring_agent.py` + `agents/assembly_collector.py`
- **AI 모델**: GPT-4o mini (의원 분석, 공략 전략 생성)
- **데이터 출처**: 국회 열린데이터 포털 (open.assembly.go.kr)
- **API 키**: `ASSEMBLY_API_KEY` 환경변수

### API 엔드포인트
| 엔드포인트 | 기능 |
|------------|------|
| `GET /api/lawmakers` | 전체 의원 목록 (정당/정렬 필터) |
| `GET /api/lawmakers/search?q=` | 의원 검색 |
| `GET /api/lawmakers/{mona_cd}` | 의원 상세 정보 |
| `POST /api/monitoring/refresh` | 국회 API 최신 데이터 수집 |
| `GET /api/monitoring/stats` | 전체 통계 |
| `GET /api/monitoring/vulnerable` | 취약 지역구 Top N |
| `GET /api/monitoring/lawmaker/{id}` | AI 의원 분석 |
| `GET /api/monitoring/strategy/{id}` | AI 공략 전략 |
| `GET /api/monitoring/report` | 종합 리포트 |
