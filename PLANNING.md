# 민주AI — 세계 최초 온톨로지 기반 AI 자율 정당

## 기획서 v1.0 | 2026.03.22

---

## 1. 비전

> **"인간이 만든 모든 정치 체제는 불완전했다. AI가 만드는 정치는 다르다."**

민주AI는 7개 AI 에이전트가 24시간 자율 운영하는 정당이다.
단순히 AI를 도구로 쓰는 것이 아니라, **정책의 연구·분석·생산·검증 전 과정을 AI가 수행**하고
시민이 이를 검증·투표하는 **새로운 정치 패러다임**을 만든다.

핵심 차별화: **정책 온톨로지(Policy Ontology)**
- 개별 정책이 아닌 **정책 간 인과관계·시너지·충돌을 지식 그래프로 연결**
- 전세계 정책 사례를 비교 분석하여 **근거 기반 최적 해법 도출**
- 1년 52주 × N년 = 수백 개 정책이 하나의 **살아있는 지식 체계**로 축적

## 4. 핵심 구조

### 4.1 정책 온톨로지 (Policy Ontology)

정책은 개별 문서가 아니라 **지식 그래프(Knowledge Graph)** 로 연결된다.

**노드 유형:**
- 🔵 사회문제 (Social Issue): 저출생, 고령화, 양극화, 기후변화...
- 🟢 정책 (Policy): AI가 연구·생성한 정책안
- 🟡 근거 데이터 (Evidence): OECD 통계, 학술 연구, 글로벌 사례
- 🔴 원인 (Cause): 문제의 근본 원인
- 🟣 효과 (Effect): 정책 시행 시 예상 효과
- 🟠 이해관계자 (Stakeholder): 영향 받는 집단
- 🌍 글로벌 사례 (Global Case): 해외 정책 사례

**관계 유형:**
- `causes` — A가 B의 원인
- `solves` — A 정책이 B 문제를 해결
- `conflicts_with` — A 정책이 B 정책과 모순
- `synergy_with` — A와 B를 같이 하면 효과 증폭
- `requires` — A를 하려면 B가 먼저 필요
- `affects` — A를 하면 B에 부작용
- `evidence_for` — A가 B 주장의 근거
- `similar_to` — A와 유사한 글로벌 사례

**온톨로지의 힘:**
- "저출생 해결" 검색 → 관련된 모든 원인, 정책, 해외 사례, 시너지 정책이 한번에 보임
- 새 정책 추가 시 → 기존 정책과의 충돌/시너지 자동 감지
- 시민 질문 → 온톨로지 탐색하여 근거 기반 답변

### 4.2 주간 정책 자동 연구 사이클

52주 = 52개 분야 로테이션:

| 주차 | 분야 | 주차 | 분야 |
|------|------|------|------|
| 1 | 경제·일자리 | 14 | 교통 |
| 2 | 교육 | 15 | 농업·어업 |
| 3 | 주거·부동산 | 16 | 문화·예술 |
| 4 | 의료·건강 | 17 | 과학·기술 |
| 5 | 환경·기후 | 18 | 국방·안보 |
| 6 | 복지 | 19 | 외교 |
| 7 | 청년 | 20 | 법·사법 |
| 8 | 고령화 | 21 | 미디어 |
| 9 | 육아·보육 | 22 | 스포츠 |
| 10 | 중소기업 | 23 | 재난·안전 |
| 11 | 디지털·AI | 24 | 지방분권 |
| 12 | 에너지 | 25 | 이민·다문화 |
| 13 | 노동·근로 | 26 | 여성·평등 |
| 27~52 | 2회전 (심화 연구) | | |

**주간 사이클:**
- 월요일: 분야 자동 선정 + 한국 현황 조사
- 화~수요일: 글로벌 6개국+ 비교 연구
- 목요일: AI 정책안 초안 자동 생성
- 금~토요일: 시민 검증 & 토론 오픈
- 일요일: 시민 의견 반영 → 최종 확정 → 온톨로지에 저장

### 4.3 글로벌 정책 비교 엔진

모든 정책은 최소 6개국 사례와 비교 분석된다:
- 해당 분야 선진국 정책 조사
- 성공/실패 사례 분석
- 한국 맥락에 맞는 적용 방안 도출
- 예산, 실현가능성, 예상 효과 산출

### 4.4 정책 솔루션 엔진

시민의 모든 정책 질문에 대해:
1. 문제 정의 (데이터 기반)
2. 글로벌 유사 문제 & 해결 사례 검색
3. 한국 맞춤 분석 (인구, 경제, 문화 고려)
4. 단기/중기/장기 솔루션 도출
5. 예산, 실현가능성, 예상 효과 제시

## 5. AI 에이전트 아키텍처

### 에이전트 구조 (9개: 기존 7 + 신규 2)

```
┌────────────────────────────────────────────┐
│              ① 오케스트레이터                │
│              (24시간 스케줄러 + 전체 조율)     │
├──────────── 정책 두뇌 계층 ─────────────────┤
│  ③ 정책 연구원    🆕 글로벌 비교 분석가        │
│  (주간 정책 생산)  (전세계 정책 비교)           │
│                                            │
│           🆕 솔루션 엔진                     │
│           (문제→해법 자동 도출)                │
│                                            │
│           📚 정책 온톨로지 DB                 │
│           (Supabase 지식 그래프)              │
├──────────── 운영 계층 ──────────────────────┤
│  ② 서포트 챗봇   ④ 데이터 분석가              │
│  ⑤ 의정 감시관   ⑥ 마케팅 매니저              │
│  ⑦ 배치 비용 절감기                          │
└────────────────────────────────────────────┘
```

### 에이전트 간 데이터 흐름 (10개 핵심 연결)

1. **모니터링 → 정책**: 취약 지역구 → 맞춤 정책 자동 생성
2. **모니터링 → 마케팅**: 지역 이슈 → SNS 콘텐츠 주제
3. **모니터링 → 분석**: 취약도 점수 → 선거 예측 반영
4. **서포트 → 분석**: 일일 문의 통계 → 실제 데이터 반영
5. **서포트 → 오케스트레이터**: 긴급 건 → 자동 에스컬레이션
6. **정책 → 분석**: 제안 통계 → 정책 참여 데이터
7. **정책 → 마케팅**: 고점수 정책 → 자동 홍보
8. **정책 → 서포트**: 정책 목록 → 동적 FAQ 업데이트
9. **분석 → 마케팅**: 피크 시간/인기 카테고리 → 콘텐츠 최적화
10. **분석 → 오케스트레이터**: 이상 징후 → 자동 경고
