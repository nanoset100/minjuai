# Design: 문의기능 (시민 → 의원 문의)
> 작성일: 2026-04-13 | 단계: Design | 참조 Plan: docs/01-plan/features/문의기능.plan.md

---

## 1. 전체 흐름

```
[프론트] 문의하기 탭
  → 주소 입력 → 의원 자동 매핑 (기존 로직 재사용)
  → 문의 내용 작성
  → [AI로 다듬기] 버튼 (선택) → POST /api/ai/polish
  → 이메일 입력 (선택)
  → [문의 보내기]
  → POST /api/letters  {letter_type: "inquiry"}
     → 스팸 필터 (GPT) → letters 테이블 저장 → SendGrid 발송
  → 확인 화면: 면책 문구 + 의원 응답률
```

## 2. DB 설계

### 2-1. letters 테이블 변경 (ALTER)

마이그레이션 파일: `migrations/016_inquiry_columns.sql`

```sql
-- 문의기능을 위한 letters 테이블 컬럼 추가
ALTER TABLE letters
  ADD COLUMN IF NOT EXISTS letter_type VARCHAR DEFAULT 'letter'
    CHECK (letter_type IN ('letter', 'inquiry')),
  ADD COLUMN IF NOT EXISTS citizen_email VARCHAR,       -- 답변 알림용 (선택, 저장만)
  ADD COLUMN IF NOT EXISTS reply_status VARCHAR DEFAULT 'pending'
    CHECK (reply_status IN ('pending', 'replied', 'no_reply')),
  ADD COLUMN IF NOT EXISTS reply_content TEXT,          -- Phase 2: 답변 내용 (비공개)
  ADD COLUMN IF NOT EXISTS reply_received_at TIMESTAMPTZ;

-- 인덱스
CREATE INDEX IF NOT EXISTS idx_letters_type ON letters (letter_type, created_at DESC);
CREATE INDEX IF NOT EXISTS idx_letters_reply_status ON letters (member_id, reply_status);
```

### 2-2. 의원 응답률 뷰 (신규)

```sql
-- 의원별 응답률 뷰 (공개 정보)
CREATE OR REPLACE VIEW member_reply_rates AS
SELECT
    member_id,
    COUNT(*) FILTER (WHERE letter_type = 'inquiry' AND status = 'sent') AS total_inquiries,
    COUNT(*) FILTER (WHERE letter_type = 'inquiry' AND reply_status = 'replied') AS total_replied,
    ROUND(
        COUNT(*) FILTER (WHERE letter_type = 'inquiry' AND reply_status = 'replied')::numeric
        / NULLIF(COUNT(*) FILTER (WHERE letter_type = 'inquiry' AND status = 'sent'), 0) * 100,
        1
    ) AS reply_rate_pct
FROM letters
GROUP BY member_id;
```

## 3. API 설계

### 3-1. POST /api/letters (기존 확장)

**Request Body 변경:**
```json
{
  "mona_cd": "VRY5522V",
  "content": "...",
  "nickname": "시민",
  "sender_district": "광주 북구갑",
  "issue_id": null,
  "letter_type": "inquiry",        // 신규: "letter" | "inquiry" (기본 "letter")
  "citizen_email": "user@email.com" // 신규: 선택 입력
}
```

**Response (문의 타입 시 추가 필드):**
```json
{
  "success": true,
  "letter_id": "uuid",
  "member_name": "민형배",
  "sent_at": "2026-04-13T09:00:00Z",
  "total_sent": 42,
  "reply_rate_pct": 23.5,           // 신규: 해당 의원 응답률
  "message": "민형배 의원실에 문의가 전달됐습니다"
}
```

### 3-2. POST /api/ai/polish (신규)

AI 다듬기 — 사용자 버튼 클릭 시에만 호출

**Request:**
```json
{
  "content": "원문 내용...",
  "type": "inquiry"   // "letter" | "inquiry"
}
```

**Response:**
```json
{
  "polished": "다듬어진 내용...",
  "original": "원문 내용..."
}
```

**GPT 프롬프트:**
```python
POLISH_PROMPT = """
다음 시민 {type_label}을 의원실 전달에 적합하게 다듬어 주세요.

원문:
{content}

규칙:
- 의미와 요청 사항은 절대 변경하지 않는다
- 비속어/감정적 표현을 정중한 표현으로 교체
- 문장 구조 자연스럽게 정리
- 200자~1000자 범위 유지
- 한국어로만 작성

다듬어진 내용만 반환 (JSON 불필요):
"""
```

### 3-3. GET /api/members/{mona_cd}/reply-rate (신규)

```json
{
  "mona_cd": "VRY5522V",
  "member_name": "민형배",
  "total_inquiries": 17,
  "total_replied": 4,
  "reply_rate_pct": 23.5
}
```

class LetterRequest(BaseModel):
    mona_cd: str
    content: str = Field(..., min_length=10)
    nickname: str = Field(default="시민", max_length=20)

### 5-4. AI 다듬기 UX 규칙
- 버튼 클릭 → `POST /api/ai/polish` 호출
- 로딩 중: 버튼 비활성화 + 스피너
- 완료 후: textarea에 다듬어진 내용 표시 + [되돌리기] 버튼 활성화
- [되돌리기]: 원문으로 복원, [되돌리기] 버튼 비활성화
- **원문은 JS 변수에 보존 (절대 덮어쓰지 않음)**
