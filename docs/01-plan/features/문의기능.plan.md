# Plan: 문의기능 (시민 → 의원 문의)
> 작성일: 2026-04-13 | 단계: Plan

---

## 1. 목표 (What & Why)

시민이 "내 지역구 의원이 내 문제를 해결해 줄 수 있나요?" 라는 질문으로
의원에게 직접 문의를 보낼 수 있는 기능을 구현한다.

**정책AI 3구조 완성:**
```
[이슈 보기] → [편지 보내기] → [문의하기]  ← 이번 구현
```

---

## 2. 합의된 핵심 스펙

### 2-1. 문의 유형
- **단일 유형**: "내 지역구 의원이 내 문제를 해결해 줄 수 있나요?"
- 편지(letter)와 동일한 DB 테이블 재사용 (`letter_type = 'inquiry'`)

### 2-2. 이메일 수집 (Option B 준비)
- **선택 입력**: 이메일 주소 필드 (의무 아님)
- 목적: 향후 답변 추적 기능(Option B) 준비
- 현재 단계: 저장만, 자동 알림 없음 (Option C 모드)

### 2-3. 확인 화면 고지
- 발송 완료 화면에 **면책 문구** 필수:
  > "의원실에서 답변이 오지 않을 수 있습니다."

### 2-4. AI 다듬기 버튼
- **사용자 트리거** (자동 변환 절대 금지)
- 버튼 클릭 시에만 GPT 호출 → 문장 다듬기
- 다듬기 전/후 원문 보존 (되돌리기 가능)

### 2-5. 답변 공개 정책
| 항목 | 공개 범위 |
|------|----------|
| 답변 받았는지 여부 | 공개 (의원별 응답률 표시) |
| 답변 내용 | 비공개 (발신자 본인만) |
| 의원별 응답률 | 공개 (WriteToThem 방식) |

---

## 3. DB 변경 계획

### 기존 `letters` 테이블에 컬럼 추가
```sql
ALTER TABLE letters
  ADD COLUMN IF NOT EXISTS letter_type VARCHAR DEFAULT 'letter'
    CHECK (letter_type IN ('letter', 'inquiry')),
  ADD COLUMN IF NOT EXISTS citizen_email VARCHAR,
  ADD COLUMN IF NOT EXISTS reply_status VARCHAR DEFAULT 'pending'
    CHECK (reply_status IN ('pending', 'replied', 'no_reply')),
  ADD COLUMN IF NOT EXISTS reply_content TEXT,
  ADD COLUMN IF NOT EXISTS reply_received_at TIMESTAMPTZ;
```

## 4. 백엔드 계획

### 4-1. 엔드포인트 재사용
- `POST /api/letters` — `letter_type` 파라미터로 구분
  - `'letter'`: 기존 편지
  - `'inquiry'`: 문의 (이번 구현)

### 4-2. 신규 엔드포인트
- `GET /api/members/{mona_cd}/reply-rate` — 의원별 응답률 조회
- `POST /api/letters/{id}/reply` — (관리자) 답변 수신 기록 (Phase 2)

### 4-3. AI 다듬기 엔드포인트
- `POST /api/ai/polish` — 편지/문의 텍스트 다듬기 (GPT-4o mini)

---

## 5. 프론트엔드 계획

### 5-1. 신규 화면: 문의하기 (`#inquiry`)
```
1. 내 주소 입력 → 의원 자동 매핑 (편지와 동일 로직)
2. 문의 내용 작성 (100자 이상)
3. [AI로 다듬기] 버튼 (선택)
4. 이메일 주소 입력 (선택, "답변 알림용 - 선택사항")
5. [문의 보내기] 버튼
```

### 5-2. 확인 화면
```
✓ 문의가 전달됐습니다
  {의원명} 의원실에 방금 문의가 전달됐어요
  {날짜 시간} | 문의 번호 #{번호}

  ⚠️ 의원실에서 답변이 오지 않을 수 있습니다.

  지금까지 {의원명} 의원 응답률: {응답률}%

  [이미지로 저장]  [다른 이슈 보기]
```

### 5-3. 하단 탭 추가
```
[이슈] [편지] [문의]   ← 문의 탭 신규 추가
```

## 6. 스팸 필터

문의에도 기존 `LETTER_FILTER_PROMPT` 동일 적용:
- 100자 미만 차단
- 욕설/비속어/혐오 표현 차단
- 스팸성 내용 차단
