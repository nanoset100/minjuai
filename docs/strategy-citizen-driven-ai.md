# 민주AI 전략: 시민 주도 AI 시스템

**작성일**: 2026-03-31
**핵심 전환**: AI가 분류한다 → 시민이 분류하고, AI가 연결한다

---

## 문제 인식

### 현재 한계

- AI 자동 매칭 정확도: 37% (3번 중 2번 틀림)
- 시민 검증 투표는 숫자만 쌓이고 활용 안 됨
- "맞아요/아니에요"만으로는 AI가 학습하지 않음

### 기존 정당과의 비교

| | 기존 정당 | 민주AI (현재) | 민주AI (전략 적용 후) |
|--|----------|-------------|-------------------|
| 방향 | 일방향 (정당→시민) | 쌍방향 시도 중 | 완전한 쌍방향 |
| 시민 역할 | 읽기, 후원 | 제보, 검증 | **의제 설정, 연대** |
| 의제 결정 | 당 지도부 | AI 자동 | **시민 직접** |
| 데이터 활용 | 거의 없음 | 수집 중 | **실시간 분석** |
| 기술 수준 | 홈페이지 | 앱 + AI | **시민 참여 AI 플랫폼** |

---

## 전략: 3단계 전환

### 1단계: 시민이 직접 이슈를 고르게 하기

**현재 흐름**:
```
제보 작성 → AI 자동 매칭 (37%) → 맞아요/아니에요
```

**변경 후 흐름**:
```
제보 작성 → AI가 후보 3~5개 추천 → 시민이 직접 선택 (또는 새 이슈 제안)
```

**왜 이게 좋은가**:
- 정확도: 37% → 90%+ (시민의 판단력)
- AI는 후보 추천만 하면 되므로 현재 수준으로 충분
- 시민이 고른 데이터가 정확한 학습 레이블이 됨
- 시민이 "내가 참여했다"는 주인의식을 느낌

**구현 난이도**: 낮음 (기존 BottomSheet UI 수정)

### 2단계: AI가 사람을 연결하기

이것이 **핵심 차별점**이다.

```
시민 A (강남구): "전세사기 심합니다" → 부동산 이슈
시민 B (송파구): "깡통전세 피해" → 부동산 이슈
시민 C (서초구): "보증금 못 돌려받음" → 부동산 이슈
                        ↓
AI: "부동산 이슈에 3개 지역구 23명이 참여 중
     함께 목소리를 모으시겠습니까?"
```

**AI가 진짜 잘하는 것**:
- 분류 (약함, 37%) ← 이걸 시민에게 넘김
- 패턴 발견 (강함) ← 이걸 AI가 함
  - 어느 지역에서 어떤 이슈가 뜨거운가
  - 이번 달 급증한 이슈는 무엇인가
  - 같은 문제를 겪는 시민은 누구인가

**기존 정당이 못 하는 이유**:
- 데이터 수집 구조가 없음 (홈페이지에 게시판이 전부)
- 시민 목소리를 실시간으로 분석할 기술이 없음
- 위에서 아래로 의제를 정하는 문화 (바꾸기 어려움)

### 3단계: 진짜 선순환

시민이 직접 고른 "제보-이슈" 쌍이 쌓이면:

```
제보 텍스트 + 시민이 고른 이슈 = 정답 레이블 (100건, 500건, 1000건)
                ↓
    한국어 특화 텍스트 분류기 학습 가능
                ↓
    1단계의 "후보 추천" 정확도 향상
                ↓
    시민이 더 빠르게 이슈 선택
                ↓
    더 많은 정확한 데이터 축적
                ↓
    ... 선순환
```

이때 비로소 "시민이 참여할수록 AI가 개선된다"가 사실이 된다.
자동이 아니라, 시민 참여가 만드는 의도적 선순환.

## 타겟 사용자

### 왜 젊은 세대에게 통하는가

1. **SNS 네이티브**: 이미 "좋아요/공유/댓글"로 의견을 표현하는 데 익숙
2. **참여형 문화**: 수동적 수용이 아니라 직접 만들고 바꾸고 싶어함
3. **데이터 감각**: 통계와 시각화를 자연스럽게 읽을 수 있음
4. **정치 불신**: 기존 정당에 실망했지만, 기술로 바꿀 수 있다는 희망

### 핵심 메시지

"당신이 제보한 문제를, AI가 같은 문제를 겪는 사람들과 연결합니다.
혼자서는 작은 목소리가, 함께 모이면 정책이 됩니다."

## 경쟁 우위 (해자/Moat)

1. **데이터 네트워크 효과**: 시민이 많아질수록 → 데이터가 쌓이고 → AI가 정확해지고 → 더 많은 시민이 오고 → ... (후발 주자가 따라잡기 어려움)
2. **시민 라벨링 데이터**: 시민이 직접 분류한 정치 텍스트 데이터는 세계적으로도 희귀 (한국어 정치 도메인)
3. **258개 이슈 온톨로지**: 전문가가 정리한 정책 체계 + 시민이 추가하는 이슈 → 살아있는 정책 사전
4. **선점 효과**: "AI 정당" 포지션을 먼저 차지하면 브랜드 가치
