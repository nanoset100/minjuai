## 앱 개요

**정책AI**는 AI가 자동 운영하는 정치 플랫폼 앱이다.
- 백엔드: FastAPI (Python) → Railway 배포
- 프론트엔드: 웹 PWA (index.html 단일 파일)
- DB: Supabase (PostgreSQL)
- AI: OpenAI GPT-4o mini

### 핵심 기능
1. 이슈맨 AI - 뉴스 RSS 자동 수집 → 지역구 매핑 → DB 등록 (하루 2회)
2. "우리 지역 문제예요!" 버튼 - AI뉴스를 시민 제보로 전환
3. 정책 제안 시스템 - 시민이 정책 제안 → AI 분석 → 투표
4. AI 상담원 - 챗봇 (정당 관련 Q&A)
5. 운영 로그 대시보드 (/logs)

## 강점 (Strengths)

### 1. 아이디어 자체는 독창적
- "AI가 운영하는 정당"이라는 개념은 세계에서 거의 없음
- 데이터 기반 정책 결정, 감정이 아닌 근거 중심 → 차별점 있음

### 2. 이슈맨 AI 자동화
- 하루 2회 뉴스 자동 수집 → DB 등록까지 완전 자동
- 8개 뉴스 소스 연동 (연합뉴스, SBS, 경향신문, 한겨레, 국민일보)
- 사람이 수동으로 할 일을 AI가 대신함

### 3. 지역구 의원 매핑
- 300개 국회의원 지역구 DB 보유
- 뉴스 → 지역구 → 담당 의원 자동 연결 시도

### 4. 시스템 기반은 탄탄함
- Supabase + Railway + FastAPI 스택 안정적
- API 엔드포인트 50개+ 구현 완료
- 정책 온톨로지 시스템 (지식 그래프) 보유

## 약점 및 솔직한 문제점 (Weaknesses)

### 1. 핵심 문제: "전국" 이슈가 너무 많음 ← 가장 심각
- 등록된 AI뉴스 중 70-80%가 district='전국'
- 지역구 매핑이 안 되면 "300명 의원 중 누구 담당?" 연결 불가
- 앱의 핵심 가치(내 지역구 의원 모니터링)가 실제로 작동 안 함
- **원인**: 대부분의 정치 뉴스는 특정 지역 언급 없이 전국 이슈로 작성됨

### 2. 실제 사용자가 없음
- Google Play Console 기준: 정책AI 앱 설치 수 0
- 앱스토어 등록도 아직 안 됨 (임시 상태)
- "시민이 직접 참여"를 강조하지만 참여할 시민이 없음
- 닭이 먼저냐 달걀이 먼저냐 문제: 사용자 없이 기능만 만들고 있음

### 3. 인증 시스템 없음
- 웹 앱에서 로그인 기능이 전혀 없음
- 시민 제보, 정책 투표가 익명으로만 가능
- 중복 제보/투표 방어가 localStorage 기반 → 우회 가능
- "당원 가입"은 이메일 수집만 하고 실제 인증은 없음

### 4. 법적/규제 리스크 불명확
- 한국에서 실제 "정당" 등록하려면 중앙선거관리위원회 신청 필요
- AI가 정당을 운영한다는 개념이 공직선거법/정당법에 위반될 수 있음
- 현재는 "정당 앱"이 아닌 "정치 정보 플랫폼"에 가까움
- 이 경계가 명확하지 않음

### 5. 프론트엔드가 단순함
- index.html 단일 파일 (800줄)로 모든 기능을 구현
- 모바일 UX가 웹 수준 → 네이티브 앱 경험 없음
- "앱스토어 앱"이 아닌 PWA라 설치/발견이 어려움

### 6. 수익 모델 없음
- 무료 서비스인데 OpenAI API 비용이 발생
- 이슈맨 AI가 하루 2회 실행 → GPT-4o mini 비용 누적
- Railway 유료 플랜 필요할 수 있음
- 지속가능성 불명확

### 7. 정책 온톨로지 활용도 낮음
- 복잡한 지식 그래프 시스템을 구축했지만
- 실제 사용자에게 보이는 UI에서 온톨로지가 노출되지 않음
- 개발 노력 대비 사용자 체감 가치가 낮음

## 외부 AI에게 묻고 싶은 질문

### Q1. 핵심 가치 명확화
"정책AI의 킬러 기능이 무엇이어야 하는가?"
지금은 뉴스 수집, 시민 제보, 정책 제안, AI 상담이 모두 있지만
어느 것도 "이것 때문에 써야 한다"는 이유가 약하다.
사용자가 정책AI를 설치하고 매일 열어볼 이유가 무엇인가?

### Q2. 전국 이슈 문제 해결 방법
지역명이 없는 뉴스를 지역구에 매핑하는 더 좋은 방법이 있는가?
예: "의원 발언" 기반 매핑, 위원회 소속 기반 매핑, 지역구 민원 키워드 DB 구축 등

### Q3. 사용자 확보 전략
0명에서 시작하는 정치 참여 앱의 초기 사용자 획득 전략은?
"좋은 기능"이 있어도 발견되지 않으면 의미없다.

### Q4. 법적 포지셔닝
"AI 운영 정당"보다 "정치 참여 플랫폼"으로 포지셔닝하는 게 맞는가?
아니면 실제 정당 등록을 추진해야 하는가?

### Q5. 우선순위 판단
지금 당장 가장 중요한 한 가지가 무엇인가?
- A) 전국 이슈 → 지역 매핑 정확도 개선 (기술)
- B) 사용자 0명 → 홍보/마케팅 (성장)
- C) 인증 시스템 추가 → 신뢰성 (기반)
- D) 특정 기능에 집중하고 나머지 제거 (선택과 집중)

### Q6. 현실적 성공 기준
6개월 후 이 앱이 "성공했다"고 볼 수 있는 최소 기준은 무엇인가?
MAU 100명? 1000명? 언론 보도? 실제 의원과의 협력?

## 개발 AI의 솔직한 총평

지금까지 기능 구현에 집중했고 기술적으로는 나쁘지 않다.
하지만 사용자가 0명인 상태에서 기능을 계속 추가하는 것이
올바른 방향인지 의문이다.

"만들고 사용자를 기다린다"가 아니라
"사용자 1명이 필요로 하는 것을 먼저 파악한다"가
더 현명한 접근일 수 있다.

가장 솔직한 한 마디:
**기능은 충분하다. 지금 필요한 건 사용자다.**
